User Guide
==========

QCANT is organized around active-space molecular workflows. Most algorithms
take the same core chemistry inputs:

- ``symbols`` and ``geometry`` for the molecule,
- ``basis``, ``charge``, and ``spin`` for the electronic-structure setup,
- ``active_electrons`` and ``active_orbitals`` for the active space,
- optional backend controls such as ``device_name`` or ``array_backend``.

Algorithm Selection
-------------------

Use ``adapt_vqe`` when you want an adaptive variational ansatz and ground-state
energy history.

Use ``qscEOM`` when you want projected excited-state spectra from a fixed ansatz
or from an ansatz returned by ``adapt_vqe`` or ``tepid_adapt``.

Use ``gcim`` when you want a projected generalized configuration-interaction
style workflow with an ADAPT-like operator selection API.

Use ``qrte``, ``qrte_pmte``, ``qkud``, or ``exact_krylov`` when you want
Hamiltonian-projected Krylov bases generated by time evolution, unitary
decomposition, or direct powers of the Hamiltonian.

Use ``cvqe`` when you want cyclic determinant growth with a compact LUCJ or
UCCSD ansatz.

Use ``tepid_adapt`` when you want a finite-temperature, ancilla-free adaptive
workflow that returns thermal weights and a qscEOM-compatible ansatz.

Use ``*_qulacs`` routines when you want CPU simulator hot-loop acceleration and
have installed Qulacs. Qulacs routines are not GPU paths in this release.

Backend Policy
--------------

QCANT keeps CPU behavior unchanged unless you explicitly request an accelerator.

PennyLane device-driven algorithms accept ``device_name`` and, where supported,
``device_kwargs``. Use ``device_name="lightning.gpu"`` for the single-GPU path.

Dense linear-algebra algorithms accept ``array_backend``:

- ``"auto"`` preserves CPU behavior unless a GPU device implies otherwise,
- ``"numpy"`` forces CPU dense arrays,
- ``"cupy"`` requests GPU dense arrays.

Sparse paths remain CPU-only in this release. See :doc:`gpu_acceleration`.

Examples And Outputs
--------------------

The top-level ``examples/`` directory contains runnable calculation inputs for
the public algorithms. Each script prints a compact summary and writes a JSON
or CSV artifact under ``examples/outputs/``.

For a quick tour:

.. code-block:: bash

   python examples/run_adapt_vqe_qsceom_h2.py
   python examples/run_krylov_family_h2.py
   python examples/run_tepid_adapt_h2.py
   python examples/run_standalone_tepid_qsceom_h2.py

For benchmark and speedup studies, see :doc:`benchmarking`.
